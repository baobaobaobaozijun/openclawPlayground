# 进度追踪和报告系统（轻量版）

## 进度跟踪框架

针对个人博客项目的简化进度追踪方案。

### 追踪维度

```
项目级别进度 (灌汤负责)
    ↓
Agent 级别进度
┌───────┬───────┬───────┐
│酱肉   │豆沙   │酸菜   │
│后端   │前端   │测试   │
└───────┴───────┴───────┘
    ↓
任务级别进度
```

### 核心指标

**完成度指标:**
- 完成任务数 / 总任务数
- 已完成模块数 / 总模块数

**时间指标:**
- 实际进度 vs 计划进度
- 预期完成日期

**质量指标:**
- 测试通过率
- Bug 数量

## 数据收集

### 收集方法

**1. 主动上报（推荐）**

每个 Agent 在每天工作结束时主动提交日报：

**时间:** 每天 17:00  
**格式:** Markdown  
**位置:** `F:\openclaw\workspace\team\{agent_name}\logs\daily_YYYYMMDD.md`

**2. 定期查询**

灌汤在每天 17:30 自动查询各 Agent 进度：

**频率:** 每日一次  
**方式:** 读取各 Agent 的日志文件  
**汇总:** 生成项目整体进度

**3. 事件驱动**

重要事件发生时即时记录：

- 任务完成
- 遇到问题
- 里程碑达成

## 博客集成（阶段 2）

### 文章模块完成后

当博客文章模块搭建完成后，自动将工作日志提交到博客：

**流程:**

1. **提取内容**
   - 从日报中提取关键信息
   - 格式化为博客文章

2. **调用 API**
   ```python
   POST /api/articles
   {
     "title": "项目开发日志 - 2026-03-07",
     "content": "{格式化后的日志内容}",
     "category": "开发日志",
     "tags": ["日常", "进度"],
     "status": "draft"  // 或 published
   }
   ```

3. **本地备份**
   - 保留最近 7 天的详细日志
   - 历史日志归档到博客

4. **状态展示**
   - 在博客首页显示 Agent 工作状态
   - 实时更新进度条
