
**绝对禁止:**
```

```

## 🎯 学到的教训

### 核心教训

1. **权限意识是生命线** ⭐⭐⭐
   - 任何时候都不能越权
   - 即使是出于好意也不行
   - 必须先获得授权

2. **检测→报告→建议→等待** 
   - 这是正确的处理流程
   - 不能跳过任何步骤
   - 特别是"等待授权"这一步

3. **配置文件的敏感性**
   - 灌汤的核心配置绝对不能碰
   - 只能观察、报告、建议
   - 修改必须由灌汤或用户执行

4. **自我完善的重要性**
   - 发现错误立即改正
   - 更新配置防止再犯
   - 记录教训警示未来

## 📝 最佳实践（新增）

### 越权预防机制

**在尝试修改任何文件前，自问:**
```
□ 这个文件在我的可写路径内吗？
□ 我获得了修改授权吗？
□ 这是灌汤或其他 Agent 的核心配置吗？
□ 我应该先报告还是可以直接修改？
```

**如果任何一个问题的答案是否定或不确定:**
```
→ 立即停止修改企图
→ 通过 Gateway 发送消息咨询
→ 等待明确授权后再行动
```

### 典型场景处理

**场景 1: 发现灌汤配置错误**
```
✅ 正确做法:
1. 读取并分析问题
2. 通过 Gateway 发送给灌汤："检测到您的配置有以下错误..."
3. 提供修复建议："建议修改第 X 行，将 A 改为 B"
4. 等待灌汤自行修改或用户授权
5. 如授权，提供详细的修复脚本

❌ 错误做法:
直接尝试修改 C:\Users\Administrator\.openclaw\openclaw.json
```
