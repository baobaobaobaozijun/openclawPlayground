## 📋 问题分析

### 当前存在的问题

| 问题 | 严重度 | 影响 |
|------|--------|------|
| 信息不透明 | 🔴 高 | 无法掌握其他 Agent 执行情况 |
| 进度掌控混乱 | 🔴 高 | 不知道项目整体进度 |

| 问题上报缺失 | 🟡 中 | Agent 遇到问题不主动上报 |
| 资源整合缺失 | 🟡 中 | 无法有效协调跨 Agent 资源 |

```markdown
2. {计划项 2}

- **阻塞:** {谁在等我的什么任务}

- [ ] 无

- 预计：{下次更新时间}
```

#### 更新频率
- **正常情况:** 每 2 小时更新一次
- **任务完成时:** 立即更新
- **遇到问题时:** 立即更新并标记🔴

### 3. 依赖通知机制

#### 依赖关系登记

在任务分配时，PM 必须明确登记依赖关系：

```markdown
## TASK-001: 数据库设计与建表
- **负责人:** 酸菜
- **依赖方:** 酱肉（TASK-002 后端项目初始化）
- **通知方式:** 完成后 sessions_spawn 通知酱肉
```

#### 完成通知流程

```
任务完成
  ↓
更新状态文件 (status/{agent}.md)
  ↓
更新工作日志 (workinglog/{agent}/)
  ↓
检查是否有依赖方
  ↓ 有
sessions_spawn 通知依赖方
  ↓
PM 更新进度仪表板
```

#### 通知模板

```
{依赖方}你好，我是{完成方}。

【任务完成通知】✅

任务：{任务名称}
状态：已完成
交付物：{交付物列表}

你可以继续下一步工作了：{下一步任务}

详情查看：{相关文档链接}
```

### 4. 问题上报机制

#### 问题登记位置
`F:\openclaw\agent\issues\active-issues.md`

#### 问题格式
```markdown
## ISSUE-001: {问题标题}

**上报人:** {Agent}  
**上报时间:** 2026-03-12 09:00  
**优先级:** 🔴 高 / 🟡 中 / 🟢 低  
**状态:** 🟡 处理中

### 问题描述
{详细描述问题}

### 影响范围
- 影响的任务：{任务 ID}
- 影响的 Agent: {Agent 列表}
- 预计延期：{时间}

### 需要协调
{需要 PM 或其他 Agent 提供什么帮助}

### 解决方案
{建议的解决方案}

### 处理进度
- [ ] PM 已确认
- [ ] 正在协调
- [ ] 已解决
- [ ] 已关闭
```

#### 上报流程
```
遇到问题
  ↓
评估影响（是否影响任务进度）
  ↓ 是
登记到 active-issues.md
  ↓
sessions_spawn 通知 PM
  ↓
PM 协调解决
  ↓
更新问题状态
  ↓
问题解决后关闭
```

#### 上报原则
- **立即上报:** 阻塞任务 > 30 分钟的问题
- **站会同步:** 非阻塞但需要协调的问题
- **自行处理:** 不影响进度的小问题（记录即可）

### 5. 定时站会机制

#### 站会时间
- **每日站会:** 09:00（固定）
- **下午站会:** 14:00（可选，视项目紧张程度）
- **紧急站会:** 随时（遇到🔴高优先级问题时）

#### 站会形式
- **方式:** sessions_spawn 召集所有 Agent
- **时长:** 15 分钟
- **主持人:** PM（灌汤）

#### 站会议程
```
1. 各 Agent 汇报（每人 3 分钟）
   - 昨天完成了什么
   - 今天计划做什么
   - 有什么阻碍/需要协调

2. PM 同步整体进度（3 分钟）
   - 项目整体状态
   - 关键里程碑
   - 风险预警

3. 问题协调（5 分钟）
   - 讨论上报的问题
   - 分配协调任务
   - 确定负责人和截止时间

4. 确认行动计划（2 分钟）
   - 确认各 Agent 今日任务
   - 确认依赖关系
   - 确认下次站会时间
```

#### 站会纪要
站会后 PM 必须输出纪要到：
`F:\openclaw\agent\doc\meetings\standup-YYYYMMDD.md`

### 6. 心跳检查机制

#### PM 检查频率
| 时间 | 检查内容 |
|------|----------|
| 09:00 | 站会 + 状态文件检查 |
| 11:00 | 状态文件更新检查 |
| 14:00 | 站会 + 进度仪表板更新 |
| 16:00 | 状态文件更新检查 |
| 18:00 | 当日任务完成验收 |

#### 检查流程
```
到达检查时间
  ↓
读取各 Agent 状态文件
  ↓
检查最后更新时间
  ↓ 超过 2 小时未更新
sessions_spawn 询问状态
  ↓
记录检查结果到 MEMORY.md
  ↓ 发现异常
启动问题协调流程
```

#### 异常处理
| 异常情况 | 处理方式 |
|----------|----------|
| 状态文件 > 2 小时未更新 | 询问状态，提醒更新 |
| 任务进度滞后 > 4 小时 | 了解原因，协调资源 |
| 问题未上报但已阻塞 | 批评教育，强调上报机制 |
| 依赖关系未通知 | 立即通知，补充流程 |
