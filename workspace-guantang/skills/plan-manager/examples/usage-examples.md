# Plan Manager 使用示例

## 示例 1: 创建新计划

**场景:** 收到新需求"用户中心模块"，需要创建计划

**命令:**
```powershell
npx plan-manager 创建计划 `
  --编号 "006" `
  --名称 "用户中心" `
  --优先级 "P1" `
  --轮次 5 `
  --负责人 "酱肉" `
  --预计耗时 "2h"
```

**输出:**
```
正在创建计划：Plan-006 - 用户中心
✓ 创建计划文件夹：plan-006-p1-用户中心
✓ 创建文档：00-plan.md
✓ 创建文档：01-round-orders.md
✓ 创建文档：02-verify-list.md
✓ 创建文档：03-review.md
✓ 创建文档：04-feishu-logs.md
✓ 填充计划元数据
✓ 更新 master-plan-overview.md
✓ 创建工作日志

✅ 计划创建完成！
计划路径：F:\openclaw\agent\tasks\01-plans\plan-006-p1-用户中心
下一步：派发第 1 轮任务给酱肉
```

## 示例 2: 更新轮次进度

**场景:** 第 1 轮任务完成，PM 验证通过

**命令:**
```powershell
npx plan-manager 更新进度 `
  --计划编号 "006" `
  --轮次 1 `
  --状态 "completed" `
  --实际耗时 "10m" `
  --备注 "提前完成"
```

**输出:**
```
正在更新进度：Plan-006 第 1 轮 - completed
✓ 更新 00-plan.md 进度表
✓ 更新 01-round-orders.md
✓ 创建工作日志

✅ 进度更新完成！
下一步：派发第 2 轮任务
```

## 最佳实践

### 1. 计划编号连续
```powershell
# ✅ 好：从 001 开始连续编号
npx plan-manager 创建计划 --编号 "001" --名称 "阻塞修复"
npx plan-manager 创建计划 --编号 "002" --名称 "文章管理"
npx plan-manager 创建计划 --编号 "003" --名称 "特色功能"

# ❌ 差：跳号
npx plan-manager 创建计划 --编号 "001" --名称 "阻塞修复"
npx plan-manager 创建计划 --编号 "005" --名称 "文章管理"  # 跳过 002-004
```

### 2. 及时更新进度
```powershell
# ✅ 好：每轮完成后立即更新
等待 Agent 完成回调 → PM 验证 → npx plan-manager 更新进度 → 派发下一轮

# ❌ 差：多轮完成后批量更新
等待 5 轮全部完成 → 一次性更新 5 轮进度  # 无法追踪中间状态
```

### 3. 必须复盘
```powershell
# ✅ 好：每个计划完成后生成复盘
npx plan-manager 完成计划 --计划编号 "006" --状态 "success"

# ❌ 差：跳过复盘
手动更新 00-plan.md 状态为 completed  # 缺少 03-review.md
```
