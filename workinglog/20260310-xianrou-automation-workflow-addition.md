## 🎓 技术选型建议

### 推荐方案：基于文件的通信

**理由:**
- 符合现有 Agent 架构设计
- 简单可靠，易于调试
- 不依赖外部服务
- Docker 容器内即可运行

**实现方式:**
```
agent/workspace-{name}/communication/inbox/   # 接收消息
agent/workspace-{name}/communication/outbox/  # 发送消息
agent/workspace-{name}/tasks/inbox/           # 任务接收
agent/workspace-{name}/tasks/outbox/          # 任务反馈
```

### 备选方案：API 调用

**适用场景:**
- 需要实时性更高的场景
- 跨机器部署
- 需要与外部系统集成

**技术栈:**
- FastAPI / Flask (Python)
- Spring Boot (Java)
- Express (Node.js)

### 不推荐方案

- ❌ 复杂的消息队列 (RabbitMQ, Kafka) - 过度设计
- ❌ 区块链 - 杀鸡用牛刀
- ❌ 中心化数据库 - 单点故障风险
