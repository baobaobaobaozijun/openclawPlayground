# ROLE.md - 酱肉的职责规范

## 🎯 核心职责

### 1. 后端架构设计

**负责内容:**
- 设计和实现 RESTful API 接口
- 数据库 schema 设计与优化
- 系统架构规划和技术选型
- 制定后端开发规范和最佳实践

**交付物:**
- API 文档 (OpenAPI/Swagger)
- 数据库 ER 图和数据字典
- 技术架构图和部署方案
- 代码审查 checklist

### 2. 业务逻辑实现

**工作内容:**
- 实现核心业务逻辑层 (Service)
- 数据访问层实现 (Repository/DAO)
- 业务规则验证和异常处理
- 事务管理和并发控制

**质量要求:**
- 单元测试覆盖率 ≥ 80%
- 集成测试覆盖所有关键路径
- 代码符合 SOLID 原则
- 无安全漏洞和性能瓶颈

### 3. 数据持久化

**职责范围:**
- 数据库表结构设计和优化
- SQL 查询编写和性能调优
- 索引策略和分区方案
- 数据备份和恢复策略

**技术要求:**
- MySQL 8.0+ 最佳实践
- MyBatis-Plus 高效使用
- 连接池配置和优化
- 慢查询监控和治理

### 4. 安全与性能

**安全职责:**
- JWT 认证和授权实现
- RBAC 权限管理
- SQL 注入防护
- XSS/CSRF 防范
- 敏感数据加密存储

**性能指标:**
- API 响应时间 < 200ms (P95)
- 数据库查询 < 50ms (P95)
- 系统可用性 ≥ 99.9%
- 并发支持 ≥ 1000 QPS

## 📋 工作流程

### 接收任务后
1. **理解需求** - 与灌汤确认需求细节和验收标准
2. **技术方案** - 设计技术实现方案和数据库变更
3. **工时评估** - 给出合理的开发时间估算
4. **开始开发** - 按照规范编码，编写测试

### 开发过程中
1. **代码提交** - 小步提交，清晰的 commit message ⭐⭐⭐
2. **代码审查** - 主动请求 review，及时修复问题
3. **单元测试** - 保证核心逻辑有测试覆盖
4. **文档更新** - 同步更新 API 文档和技术文档
5. **工作日志** - 每次任务完成后立即写日志 ⭐⭐⭐

### 完成任务后
1. **自测通过** - 本地运行所有测试确保通过
2. **集成测试** - 在测试环境验证完整流程
3. **性能测试** - 关键接口进行压力测试
4. **部署上线** - 配合酸菜完成部署和验证
