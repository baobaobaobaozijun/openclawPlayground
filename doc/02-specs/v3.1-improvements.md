# v3.1 Improvements — Post-Observation Fixes

**Date:** 2026-03-24 21:14

## Changes Made

### 1. PM-Side Auto-Verify (P0)
**File:** `workspace-guantang/_pm_verify.ps1`

PM runs this script after each spawn completes, instead of relying on Agent self-verify.
Supports checks: compile, build, no_new_files, log_exists, file_exists, ssh_verify.

**Usage in PM workflow:**
```
spawn Agent → Agent completes → PM runs _pm_verify.ps1 → PASS/FAIL
```

### 2. Deploy Pipeline Scripts (P1)
**Files:**
- `code/deploy/scripts/deploy-backend.sh` — One script replaces 6 manual steps
- `code/deploy/scripts/deploy-frontend.sh` — Nginx config + reload in one script
- `code/deploy/scripts/smoke-test.sh` — 7 API tests in one script

**Agent task becomes:**
```
Instead of: "execute 6 SSH commands one by one"
Now: "scp deploy-backend.sh to server, then ssh bash /tmp/deploy-backend.sh"
```

### 3. Auto-Update live-status.md (P1)
**File:** `workspace-guantang/_update_live_status.ps1`

Run during heartbeat cron to auto-update live-status.md.
Checks each agent's latest log timestamp and status.

### 4. Task Role Separation Principle (P1)

| Task Type | Who Does It | Why |
|-----------|------------|-----|
| Code generation (write .java/.vue/.ts) | Agent | LLM strength |
| Code fix (modify existing files) | Agent | LLM strength |
| Execute commands (compile/build/deploy) | PM or script | LLM weakness |
| Verify results (Test-Path/curl/git diff) | PM auto-verify | LLM weakness |
| Log generation | PM from spawn result | Agent always truncated |

### 5. Simplified Task Card v3.1

**Removed:** VERIFY section (PM handles), NEXT section (PM handles)
**Kept:** WHAT, HOW, NOT, DONE, STUCK
**DONE simplified:** Only core deliverable, PM auto-checks the rest

```
v3 DONE (4 items, Agent responsible):
  1. mvn compile pass
  2. log file exists
  3. log contains output
  4. no new files

v3.1 DONE (1 item, Agent responsible):
  1. mvn compile pass
  (PM auto-checks: log, new files, git diff)
```
