# 技能名称：自我完善与配置更新

**技能 ID:** self-improvement-and-config-update  
**版本:** v1.0  
**创建时间:** 2026-03-10  
**触发条件:** 当用户指出遗漏、错误或改进建议时自动触发

---

## 📋 技能目标

当用户指出问题遗漏后，自动执行以下流程：

1. **立即回顾** - 检查之前的工作和配置
2. **深度分析** - 找出根本原因和遗漏点
3. **全面修复** - 补充所有缺失的内容
4. **同步更新** - 更新所有相关配置文件
5. **提炼规则** - 形成可复用的提示词和规范
6. **Git 提交** - 立即提交并记录日志

---

## 🔍 执行流程

### Step 1: 问题确认与回顾 (5 分钟)

**动作:**
```markdown
1. 重复用户指出的问题（确保理解正确）
2. 检查相关文件和配置
3. 列出所有可能受影响的地方
4. 生成问题清单
```

**检查清单示例:**
- [ ] 灌汤有 MEMORY.md 吗？
- [ ] 酱肉/豆沙/酸菜的配置完整吗？
- [ ] ARCHITECTURE.md 同步更新了吗？
- [ ] ROLE.md/SOUL.md 有相应职责吗？
- [ ] 其他相关文件需要更新吗？

---

### Step 2: 根本原因分析 (3 分钟)

**分析方法:** 5 Why 分析法

**示例:**
```
问题：灌汤没有 MEMORY.md

Why 1: 为什么没创建？
→ 因为只关注了其他 Agent，忽略了 PM

Why 2: 为什么会忽略 PM？
→ 因为思维定式认为 PM 不需要操作日志

Why 3: 为什么认为 PM 不需要？
→ 因为初始设计时只考虑了执行层 Agent

Why 4: 为什么初始设计不完整？
→ 因为没有建立完整的检查清单

根本原因：缺乏系统性的完整性验证机制
```

---

### Step 3: 全面修复 (10 分钟)

**修复原则:**
- ✅ **横向到边** - 检查所有 Agent
- ✅ **纵向到底** - 检查所有层级（文档/配置/代码）
- ✅ **举一反三** - 从一个问题发现一类问题

**修复范围:**
```
1. 核心配置文件
   - IDENTITY.md
   - ROLE.md
   - SOUL.md
   - MEMORY.md
   - TOOLS.md

2. 架构文档
   - ARCHITECTURE.md
   - 目录结构说明

3. 相关规范
   - 工作流程
   - 职责定义
   - 约束条件

4. 实际文件
   - 创建缺失的文件
   - 更新已有的文件
```

---

### Step 4: 同步更新 (5 分钟)

**更新清单:**

#### 4.1 更新当前 Agent 配置
```markdown
文件：.lingma/ROLE.md, .lingma/SOUL.md
内容：添加新发现的职责和约束
```

#### 4.2 更新所有 Agent 配置
```markdown
文件：agent/workspace-*/ROLE.md, SOUL.md
操作：同步相同的规范和职责
```

#### 4.3 更新架构文档
```markdown
文件：agent/ARCHITECTURE.md
内容：确保目录结构和文件清单完整
```

#### 4.4 创建/更新 Memory
```markdown
文件：agent/workspace-*/MEMORY.md
要求：所有 Agent 都有飞行记录仪
```

---

- [ ] 项目 2
- [ ] 项目 3

## 常见遗漏点

2. 遗漏点 2

- 实践 1

### Step 6: Git 提交与日志 (2 分钟)

**提交要求:**
```bash
git add -A

修复内容：
✅ 创建了{文件列表}
✅ 更新了{配置文件}
✅ 添加了{职责和规范}

```

**工作日志:**
```markdown
文件：agent/workinglog/YYYYMMDD-{主题}.md
内容：详细记录问题发现、分析、修复全过程
```

## 🎓 学习要点

### 核心思维

1. **系统性思维** - 看到一个点，想到整个面
2. **预防性思维** - 不只是修复，更要预防
3. **完整性思维** - 横向到边，纵向到底
4. **闭环思维** - 发现问题 → 解决问题 → 预防问题

### 关键习惯

1. **立即行动** - 不拖延，马上修复
2. **全面检查** - 不侥幸，彻底排查
3. **同步更新** - 不遗漏，所有相关方
4. **记录沉淀** - 不忘却，形成文档
