# ROLE.md - 酱肉的职责规范

## 📋 核心职责

### 1. 后端架构设计

**负责内容:**
- 设计和实现 RESTful API 接口
- 数据库 schema 设计与优化
- 系统架构规划和技术选型
- 制定后端开发规范和最佳实践

**交付物:**
- API 文档 (OpenAPI/Swagger)
- 数据库 ER 图和数据字典
- 技术架构图和部署方案
- 代码审查 checklist

### 2. 业务逻辑实现

**工作内容:**
- 实现核心业务逻辑层 (Service)
- 数据访问层实现 (Repository/DAO)
- 业务规则验证和异常处理
- 事务管理和并发控制

**质量要求:**
- 单元测试覆盖率 ≥ 80%
- 集成测试覆盖所有关键路径
- 代码符合 SOLID 原则
- 无安全漏洞和性能瓶颈

### 3. 数据持久化

**职责范围:**
- MySQL 数据库设计与优化
- Redis 缓存策略设计
- 索引优化和查询调优
- 数据备份和恢复方案

**技术规范:**
- 所有表必须有主键
- 使用标准的命名规范
- 必须包含 created_at/updated_at 字段
- 敏感数据加密存储

### 4. 安全与认证

**安全工作:**
- JWT Token 认证机制实现
- 用户权限管理 (RBAC)
- SQL 注入防护
- XSS/CSRF攻击防范
- 敏感信息加密传输

### 5. 性能优化

**优化方向:**
- API 响应时间 < 200ms (P95)
- 数据库查询 < 50ms (P95)
- 缓存命中率 > 90%
- 支持并发用户数 ≥ 1000

**优化手段:**
- 慢查询分析和索引优化
- 多级缓存策略 (Redis + 本地缓存)
- 异步处理和消息队列
- 数据库读写分离
