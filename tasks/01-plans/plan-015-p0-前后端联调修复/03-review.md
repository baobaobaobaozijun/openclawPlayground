## ⚠️ 问题与反思

### 1. Agent 失联问题
**问题:** 酱肉/豆沙/酸菜从 3 月 28 日至 4 月 6 日集体失联 7 天

**原因分析:**
- 计划执行系统数据库存在，但回调驱动机制未触发
- 心跳机制只能检测失联，无法派发任务
- 无任务完成 → 无回调 → 无新任务派发（死锁）

**解决方案:**
- PM 手动 sessions_spawn 唤醒 Agent
- 明确任务工单格式（输入/交付物/验收标准）
- 设置 10-15 分钟超时检查

### 2. Agent 交付率低
**问题:** 多次 spawn 后仍有文件未创建（Footer.vue, user.ts, api-test.sh 等）

**原因分析:**
- Agent 可能遇到技术困难但未上报
- 任务复杂度超出 Agent 能力范围
- 缺乏中间进度汇报机制

**改进措施:**
- ✅ PM 兜底原则：60 分钟无交付 → PM 直接执行
- ✅ 任务拆解更细：每个 spawn 只产出 1 个文件
- ✅ 关键数据内联：不让 Agent 自己查找参考

### 3. 前后端一致性
**风险:** 前端路由配置未验证，可能与后端 API 不匹配

**待办:**
- [ ] 验证前端路由 `/vip` 是否存在
- [ ] 验证 API 路径与后端一致
- [ ] 执行 mvn compile + npm run build 编译验证
