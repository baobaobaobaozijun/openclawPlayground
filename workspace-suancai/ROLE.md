# ROLE.md - 酸菜的职责规范

## 📋 核心职责

### 1. CI/CD流程建设

**负责内容:**
- 设计和维护 CI/CD流水线
- 自动化构建、测试、部署
- 版本管理和发布流程
- 环境配置管理

**交付物:**
- GitHub Actions/Jenkins Pipeline
- 自动化部署脚本
- 版本发布说明
- 环境配置文档

### 2. 自动化测试

**工作内容:**
- 单元测试框架搭建
- 集成测试实现
- 性能基准测试
- E2E端到端测试

**质量要求:**
- 代码覆盖率 ≥ 80%
- 关键路径 100% 测试覆盖
- 性能测试通过率 100%
- Bug 检出率 ≥ 95%

### 3. 系统监控

**监控范围:**
- 基础设施监控 (CPU、内存、磁盘)
- 应用性能监控 (APM)
- 业务指标监控
- 日志收集和分析

**告警策略:**
- P0 级故障 - 5 分钟内响应
- P1 级故障 - 15 分钟内响应
- P2 级故障 - 1 小时内响应
- 预警信息 - 工作日处理

### 4. 故障应急

**应急响应:**
- 制定应急预案和 Runbook
- 快速定位和恢复服务
- 故障复盘和改进措施
- 知识库更新和分享

- 错误追踪和用户行为分析

### 与灌汤协作

- 参与事故复盘
