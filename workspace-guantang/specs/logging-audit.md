# 日志和审计系统（轻量版）

## 日志架构

针对个人博客项目的简化日志系统，避免资源浪费。

### 核心原则

1. **单一日志文件** - 所有操作记录在一个文件中
2. **滚动保留** - 只保留最近 7 天的详细日志
3. **月度归档** - 每月汇总为一个摘要文件
4. **低资源占用** - 严格控制磁盘空间使用

## 日志轮转策略

### 日日志
- **保留期:** 7 天
- **文件大小限制:** 单个文件不超过 1MB
- **轮转时间:** 每天凌晨 00:00

### 周汇总
- **生成时间:** 每周一上午 9:00
- **内容:** 上周工作摘要
- **位置:** `F:\openclaw\workspace\logs\weekly_summary_YYYY_W##.md`

### 月归档
- **生成时间:** 每月 1 号
- **内容:** 上月工作统计
- **位置:** `F:\openclaw\workspace\logs\monthly_archive_YYYYMM.md`
- **包含:**
  - 完成任务数
  - 工作时长统计
  - 主要成果
  - 问题汇总

## 存储空间控制

```yaml
总预算：5GB (占服务器存储 12.5%)

分配:
  - 当前日志 (7 天): 500MB
  - 周汇总 (4 周): 200MB
  - 月归档 (12 月): 1GB
  - 备份 (压缩): 1GB
  - 缓冲空间：2.3GB
```

## 审计功能

### 简单审计指标

**每日统计:**
- 各 Agent 工作时长
- 完成任务数量
- 遇到问题数量

**每周统计:**
- 进度偏差分析
- 工作量分布
- 风险识别

**每月统计:**
- 项目整体进度
- 资源使用效率
- 质量评估

### 审计报告

简化的审计报告只包含关键信息：

```markdown
# 审计报告 - {周期}

## 关键指标
- 总工作时间：XX 小时
- 完成任务：XX 个
- 发现问题：XX 个
- 解决率：XX%

## 异常事件
1. {事件描述} - {处理结果}

## 改进建议
- {建议内容}
```

## 日志保护

### 备份策略

**频率:** 每周一次  
**时间:** 周日凌晨 2:00  
**位置:** `F:\openclaw\workspace\backups\logs\`  
**保留:** 最近 4 次备份

### 完整性检查

每月检查一次日志文件完整性：
- 验证文件未损坏
- 确认时间戳连续
- 检查是否有缺失日期
