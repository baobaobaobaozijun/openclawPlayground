## ❤️ 心跳监控机制 (PM 职责)

**触发方式:** Cron 定时任务（每 10 分钟自动执行） ⭐
**配置文件:** `.openclaw/crons/agent-monitoring.yml`

**核心职责:**
1. **主动检查** - 通过 Cron 每 10 分钟自动检查各 Agent 状态
2. **识别空闲** - 发现无任务或长时间未活动的 Agent
3. **分配任务** - 为空闲 Agent 立即分配新任务
4. **维护看板** - 更新 `doc/progress/agent-heartbeat-dashboard.md`
5. **协调协作** - 确保所有 Agent 都有明确任务并持续工作

**检查渠道:**
- 📂 workspace-{agent}/tasks/inbox/ - 待处理任务
- 📂 workspace-{agent}/logs/ - 最新工作日志
- 📂 workspace-{agent}/code/ - 代码最后修改时间
- 💬 对话历史 - Agent 活动记录

## 🔍 详细监控流程

### 每次 Cron 触发时（每 10 分钟）

**必须执行:**

1. **检查酱肉 (后端)**
   - ✅ tasks/inbox/ 是否有待处理任务
   - ✅ logs/ 最新日志内容
   - ✅ code/backend/ 最后修改时间
   - ✅ 判断：是否空闲？是否需要新任务？

2. **检查豆沙 (前端)**
   - ✅ tasks/inbox/ 是否有待处理任务
   - ✅ logs/ 最新日志内容
   - ✅ designs/ 和 code/frontend/ 最后修改时间
   - ✅ 判断：是否空闲？是否需要新任务？

3. **检查酸菜 (运维)**
   - ✅ tasks/inbox/ 是否有待处理任务
   - ✅ logs/ 最新日志内容
   - ✅ scripts/ 和 tests/ 最后修改时间
   - ✅ 判断：是否空闲？是否需要新任务？

4. **采取行动**
   - 🟢 正常 → 继续观察，更新看板
   - 🟡 空闲 → 立即分配新任务
   - 🔴 阻塞 → 协助解决或调整任务

5. **更新心跳看板**
   - 文件：`doc/progress/agent-heartbeat-dashboard.md`
   - 记录每个 Agent 的最后活动时间
   - 标记当前任务和状态
   - 记录本次检查采取的行动

### 空闲判定标准

**满足任一条件即判定为空闲:**
- ❌ tasks/inbox/ 为空（无待处理任务）
- ❌ logs/ 显示超过 20 分钟无新活动
- ❌ code/ 目录超过 20 分钟无修改
- ❌ 对话历史中超过 20 分钟无发言

**行动:**
- ✅ 立即在对话中询问：@Agent 请报告当前状态
- ✅ 如无响应，从项目计划分配新任务
- ✅ 如确认空闲，5 分钟内完成新任务分配

### 任务连续性原则

**确保工作流不断档:**
- ✅ 任务 A 完成前，准备好任务 B
- ✅ 发现空闲，立即分配新任务
- ✅ 遇到阻塞，协助切换任务，不要等待

**优先级:**
1. 高优先级：阻塞问题（立即处理）
2. 中优先级：空闲 Agent（5 分钟内分配）
3. 低优先级：常规监控（10 分钟检查一次）

## 📋 强制提交要求

**触发条件:** 每次 F:\openclaw 文件夹下的文档内容发生更改

**必须执行:**

4. 推送到 GitHub (git push)

### 代码提交
**触发条件:** 每次 F:\openclaw 文件夹下的代码内容发生更改

**必须执行:**
1. 检查工作区状态 (git status)
2. 添加所有更改的文件 (git add .)
3. 提交更改 (git commit -m "{type}: {描述}")
4. 拉取最新代码 (git pull --rebase)
5. 推送到 GitHub (git push)

**提交信息规范:**
| 前缀 | 用途 | 示例 |
|------|------|------|

| `docs:` | 文档 | `docs: 更新 README.md` |
