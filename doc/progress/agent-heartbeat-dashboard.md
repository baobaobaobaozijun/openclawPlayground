## 📝 使用说明

### 如何更新此看板

**灌汤（PM）职责:**

1. **每次 Cron 触发后** (每 10 分钟):
   - 更新"实时状态总览"表格
   - 填写"本轮检查详情"各 Agent 部分
   - 如有异常，记录到"异常情况记录"

2. **每日结束时**:
   - 更新"统计分析 - 今日汇总"
   - 清理已解决的异常记录

3. **每周结束时**:
   - 更新"统计分析 - 本周趋势"
   - 总结"改进行动"进度

### 状态判定标准

**🟢 正常工作中:**
- 有活跃会话 OR
- 过去 30 分钟内有 Git 提交 OR
- 过去 20 分钟内有日志更新

**🟡 空闲待命:**
- 无活跃会话 AND
- 无 Git 提交 AND
- inbox 为空（无待处理任务）

**🔴 异常停滞:**
- 无活跃会话 AND
- 有待处理任务但未执行 AND
- 超过 60 分钟无活动

### 响应时限要求

- **Agent 响应心跳:** ≤ 5 分钟
- **PM 分配新任务:** ≤ 10 分钟（发现空闲后）
- **异常问题处理:** ≤ 30 分钟
