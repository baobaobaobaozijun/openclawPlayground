## 1. 上线概述

### 1.1 上线目标
将博客系统部署到阿里云生产环境，确保系统稳定运行。

### 1.2 上线范围
**包含:**
- 用户认证模块（手机号登录、JWT Token）
- 文章管理模块（CRUD、分类、标签）
- 权限控制（公开/登录/VIP）
- 日志自动化（定时任务）

**不包含:**
- 评论系统（阶段 2）
- 付费 VIP 功能（阶段 2）

## 4. 回滚方案

### 4.1 回滚触发条件

| 条件 | 说明 | 决策人 |
|------|------|--------|
| 核心功能不可用 | 登录、文章等核心功能故障 | PM |
| 严重性能问题 | 响应时间 > 5 秒 | 运维 |
| 数据异常 | 数据丢失或错误 | 技术负责人 |
| 验证不通过 | 验证测试失败率 > 20% | 测试 |

### 4.2 回滚步骤

#### 步骤 1: 停止服务

```bash
cd /opt/blog-system
docker compose down
```

#### 步骤 2: 恢复数据库

```bash
# 恢复备份
mysql -u root -p blog_system < /backup/blog_system_20260326_220000.sql
```

#### 步骤 3: 恢复旧版本

```bash
# 切换回旧版本
git checkout {旧版本 commit}

# 重新构建
docker compose build
docker compose up -d
```

#### 步骤 4: 验证回滚

```bash
# 验证服务正常
curl http://localhost:8080/api/health
```

**回滚负责人:** 酸菜
