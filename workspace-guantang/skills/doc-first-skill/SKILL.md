# 文档优先技能 (doc-first-skill)

## 用途

**核心目标:** 确保"文档优先"原则落地执行

**使用场景:**
1. 收到新需求时，快速创建 PRD 文档
2. 分配任务时，自动附带文档链接
3. 检查文档创建状态
4. 通知相关 Agent 阅读文档

---

## 快速创建文档

### 命令格式

```bash
npx doc-first create --type {文档类型} --project {项目名} --pm {负责人}
```

### 文档类型

| 类型 | 模板 | 输出位置 | 时限 |
|------|------|----------|------|
| `prd` | PRD 模板.md | doc/prd/{项目名}-PRD.md | 1 小时 |
| `tech` | API 技术方案模板.md | doc/tech/{项目名}-技术方案.md | 2 小时 |
| `design` | 原型交互稿模板.md | doc/design/{项目名}-原型交互稿.md | 2 小时 |
| `test` | 测试用例模板.md | doc/test/{项目名}-测试用例.md | 开发完成前 |
| `deploy` | 上线方案模板.md | doc/deploy/{项目名}-上线方案.md | 上线前 4 小时 |
| `plan` | 项目进度计划模板.md | doc/plan/{项目名}-项目计划.md | 1 小时 |

### 示例

```bash
# 创建 PRD 文档
npx doc-first create --type prd --project 博客系统 --pm 灌汤

# 创建技术方案
npx doc-first create --type tech --project 博客系统 --pm 酱肉

# 创建项目计划
npx doc-first create --type plan --project 博客系统 --pm 灌汤
```

## 任务分配模板

### 标准格式

```markdown
{Agent} 你好，

【任务分配】{任务名称}

📋 相关文档:
- PRD: [doc/prd/{项目名}-PRD.md](链接)
- 技术方案：[doc/tech/{项目名}-技术方案.md](链接)
- 原型稿：[doc/design/{项目名}-原型交互稿.md](链接)
- 项目计划：[doc/plan/{项目名}-项目计划.md](链接)

请阅读文档后确认：
1. 已阅读 PRD 文档
2. 已阅读技术方案
3. 已知晓任务内容
4. 预计完成时间

---
灌汤 | PM 🍲
```

### 使用方式

```bash
npx doc-first assign --agent {AgentID} --task {任务名称} --docs {文档列表}
```

### 示例

```bash
# 分配任务给酱肉
npx doc-first assign \
  --agent jiangrou \
  --task "后端 API 开发" \
  --docs "doc/prd/博客系统-PRD.md,doc/tech/博客系统 - 技术方案.md"

# 分配任务给豆沙
npx doc-first assign \
  --agent dousha \
  --task "前端页面开发" \
  --docs "doc/prd/博客系统-PRD.md,doc/design/博客系统 - 原型交互稿.md"
```

## 最佳实践

### 1. 文档先行

**✅ 正确做法:**
```
收到需求 → 创建 PRD → 评审 → 创建技术方案 → 评审 → 开始开发
```

**❌ 错误做法:**
```
收到需求 → 开始开发 → 补文档
```

### 2. 链接有效

**✅ 正确做法:**
- 使用相对路径：`doc/prd/博客系统-PRD.md`
- 使用绝对链接：`https://github.com/.../doc/prd/博客系统-PRD.md`
- 定期验证链接有效性

**❌ 错误做法:**
- 使用本地路径：`F:\openclaw\agent\doc\prd\...`
- 使用过期链接
- 使用无效链接

### 3. 及时更新

**✅ 正确做法:**
- 文档变更时，立即通知相关 Agent
- 使用版本号管理文档
- 保留修订历史

**❌ 错误做法:**
- 文档更新后不通知
- 多个版本文档并存
- 无修订历史

## 故障排除

### 问题 1: 文档创建失败

**原因:** 模板文件不存在

**解决:**
```bash
# 检查模板是否存在
ls doc/templates/

# 如不存在，重新创建模板
npx doc-first init-templates
```

### 问题 2: 文档链接无效

**原因:** 路径错误或文件未提交

**解决:**
```bash
# 验证文件是否存在
ls doc/prd/{项目名}-PRD.md

# 提交文件
git add doc/prd/{项目名}-PRD.md
git commit -m "docs: 创建 PRD 文档"
git push
```

### 问题 3: Agent 未收到文档

**原因:** 通知未发送或发送失败

**解决:**
```bash
# 重新发送通知
npx doc-first assign --agent {AgentID} --task {任务名称} --docs {文档列表} --force
```
