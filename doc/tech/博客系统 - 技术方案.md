## 1. 概述

### 1.1 背景
构建轻量级博客系统，支持 Agent 团队工作日志自动提交，实现开发过程透明化。

### 1.2 目标
- 技术方案目标：RESTful API + JWT 认证 + Docker 容器化
- 性能目标：API 响应时间 P95 < 200ms，页面加载 < 2 秒
- 质量目标：单元测试覆盖率 ≥ 80%

### 1.3 范围
**包含:**
- 用户认证模块（手机号登录、JWT Token）
- 文章管理模块（CRUD、分类、标签）
- 权限控制（公开/登录/VIP）
- 日志自动化（定时任务）

**不包含:**
- 评论系统（阶段 2）
- 付费 VIP 功能（阶段 2）

## 4. API 设计

### 4.1 API 清单

| 接口名称 | 方法 | 路径 | 权限 | 负责人 |
|----------|------|------|------|--------|
| 用户登录 | POST | /api/auth/login | 公开 | 酱肉 |
| 文章列表 | GET | /api/articles | 公开 | 酱肉 |
| 文章详情 | GET | /api/articles/{id} | 公开 | 酱肉 |
| 创建文章 | POST | /api/articles | 登录 | 酱肉 |
| 更新文章 | PUT | /api/articles/{id} | 作者 | 酱肉 |
| 删除文章 | DELETE | /api/articles/{id} | 作者/管理员 | 酱肉 |
| 分类列表 | GET | /api/categories | 公开 | 酱肉 |
| 标签列表 | GET | /api/tags | 公开 | 酱肉 |

### 4.2 接口详情

#### 接口 1: 用户登录

**请求:**
```http
POST /api/auth/login
Content-Type: application/json

{
  "phone": "13800138000",
  "password": "password123"
}
```

**响应:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "phone": "13800138000",
      "nickname": "用户昵称",
      "avatar": "https://..."
    }
  },
  "timestamp": 1710234567890
}
```

**错误码:**
| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| 400 | 参数错误 | 检查请求参数 |
| 401 | 认证失败 | 重新登录 |
| 403 | 无权限 | 联系管理员 |
| 404 | 资源不存在 | 检查资源 ID |
| 500 | 服务器错误 | 联系后端 |

## 5. 数据库设计

### 5.1 ER 图

```
users (用户表)
  │
  │ 1:N
  ▼
articles (文章表)
  │
  │ N:M
  ▼
tags (标签表)
  
articles (文章表)
  │
  │ N:1
  ▼
categories (分类表)
```

### 5.2 表结构

#### 表 1: users (用户表)

| 字段 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| id | bigint | - | 是 | 自增 | 主键 |
| phone | varchar | 11 | 是 | - | 手机号 |
| password_hash | varchar | 255 | 是 | - | 密码哈希 |
| nickname | varchar | 50 | 否 | - | 昵称 |
| avatar | varchar | 255 | 否 | - | 头像 |
| role | varchar | 20 | 是 | user | 角色 |
| created_at | datetime | - | 是 | NOW() | 创建时间 |

**索引:**
- `idx_phone` (phone) - 手机号唯一索引

#### 表 2: articles (文章表)

| 字段 | 类型 | 长度 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| id | bigint | - | 是 | 自增 | 主键 |
| user_id | bigint | - | 是 | - | 作者 ID |
| title | varchar | 200 | 是 | - | 标题 |
| content | text | - | 是 | - | Markdown 内容 |
| html_content | text | - | 是 | - | HTML 内容 |
| status | varchar | 20 | 是 | draft | 状态 |
| access_level | varchar | 20 | 是 | public | 访问级别 |
| view_count | int | - | 否 | 0 | 阅读量 |
| created_at | datetime | - | 是 | NOW() | 创建时间 |
| updated_at | datetime | - | 是 | NOW() | 更新时间 |

**索引:**
- `idx_user_id` (user_id)
- `idx_status` (status)
- `idx_created_at` (created_at)

## 7. 非功能设计

### 7.1 性能优化

| 优化点 | 方案 | 预期效果 |
|--------|------|----------|
| 数据库查询 | 添加索引 | 查询时间 < 50ms |
| 接口响应 | Redis 缓存 | 响应时间 < 200ms |
| 前端加载 | 懒加载 + 代码分割 | 首屏 < 2 秒 |

### 7.2 安全设计

- [x] JWT Token 认证
- [x] 密码加密存储 (bcrypt)
- [x] SQL 注入防护 (参数化查询)
- [x] XSS 防护 (输入过滤)
- [x] CSRF 防护 (Token 验证)

### 7.3 容错设计

| 场景 | 容错方案 |
|------|----------|
| 数据库连接失败 | 重试 3 次，降级返回缓存 |
| 接口超时 | 超时时间 5 秒，友好提示 |
| 服务宕机 | 健康检查 + 自动重启 |
