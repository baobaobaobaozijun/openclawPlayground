## 项目概述

### 项目背景

构建一个轻量级个人博客系统，支持 Agent 团队工作日志自动提交，实现开发过程透明化和记录。

### 项目目标

1. **文章管理** - 支持 Markdown 格式，分类标签管理
2. **用户认证** - 手机号登录，分级权限控制
3. **日志自动化** - 每日 18:00 自动将 Agent 工作日志发布为博客文章
4. **工作状态展示** - 实时展示 4 个 Agent 的工作进度

### 项目范围

**包含:**
- 用户认证模块 (手机号登录)
- 文章管理模块 (Markdown 编辑)
- 前端展示页面 (访客模式 + 管理后台)
- 日志自动提交定时任务
- Docker 部署环境

**不包含:**
- 评论系统 (阶段 2)
- 付费 VIP 功能 (阶段 2)
- 多语言支持 (阶段 3)

## 功能需求

### FR-1: 用户认证模块

| ID | FR-1 |
|----|------|
| **功能名称** | 用户登录/注册 |
| **优先级** | P0 (最高) |
| **描述** | 用户通过手机号登录，系统自动判断是否注册 |
| **输入** | 11 位手机号码 |
| **处理逻辑** | 1. 验证手机号格式 `^1[3-9]\d{9}$`<br>2. 查询数据库是否存在<br>3. 已存在：生成 JWT Token<br>4. 不存在：自动创建用户 + 生成 JWT Token |
| **输出** | JWT Token + 用户信息 |
| **验收标准** | - 符合格式的手机号均可登录<br>- 无需验证码<br>- Token 有效期 2 小时 |

---

### FR-2: 访客模式

| ID | FR-2 |
|----|------|
| **功能名称** | 访客浏览 |
| **优先级** | P0 |
| **描述** | 未登录用户可访问公开内容 |
| **输入** | 无 |
| **处理逻辑** | 1. 检测用户是否登录<br>2. 未登录：仅展示 access_level=0 的内容<br>3. 访问受限内容时弹出登录提示 |
| **输出** | 公开内容或登录提示 |
| **验收标准** | - 首页文章列表无需登录<br>- 公开文章详情无需登录<br>- 受限内容正确拦截 |

---

| ID | FR-3 |
|----|------|
| **功能名称** | 文章 CRUD |
| **优先级** | P0 |
| **描述** | 创建、读取、更新、删除文章 |
| **输入** | 文章标题、Markdown 内容、分类、标签、访问级别 |
| **处理逻辑** | 1. 创建文章时存储 Markdown 原文<br>2. 后端渲染为 HTML 供前端展示<br>3. 支持文章级别设置 (公开/登录/VIP) |
| **输出** | 文章对象 (含 HTML 渲染结果) |
| **验收标准** | - 支持 Markdown 语法<br>- 支持代码高亮<br>- 支持表格、流程图 |

### FR-6: 工作日志自动提交

| ID | FR-6 |
|----|------|
| **功能名称** | 定时任务自动提交 |
| **优先级** | P1 |
| **描述** | 每日 18:00 自动抓取 Agent 工作日志并发布为博客文章 |
| **输入** | 无 (定时触发) |
| **处理逻辑** | 1. 每日 18:00 触发定时任务<br>2. 遍历 4 个 Agent 的日志目录<br>3. 读取当日日志文件<br>4. 解析为标准博客格式<br>5. 创建文章 (状态：已发布)<br>6. 记录提交日志 |
| **输出** | 博客文章 + 提交记录 |
| **验收标准** | - 每日 18:00 准时执行<br>- 4 个 Agent 日志全部处理<br>- 失败有错误日志 |

---

### FR-7: Agent 状态展示

| ID | FR-7 |
|----|------|
| **功能名称** | Agent 工作状态页 |
| **优先级** | P1 |
| **描述** | 展示 4 个 Agent 的实时工作状态 |
| **输入** | 无 (前端轮询) |
| **处理逻辑** | 1. 前端每 30s 轮询状态 API<br>2. 后端返回 Agent 状态信息<br>3. 前端渲染状态卡片 |
| **输出** | Agent 状态列表 |
| **验收标准** | - 状态实时更新<br>- 进度条显示正确 |

---

### FR-8: 权限控制

| ID | FR-8 |
|----|------|
| **功能名称** | 分级权限控制 |
| **优先级** | P0 |
| **描述** | 文章和模块的访问级别控制 |
| **输入** | 用户角色 + 资源访问级别 |
| **处理逻辑** | 1. 定义 3 级访问级别<br>2. 中间件检查用户权限<br>3. 无权限时返回 403 或登录提示 |
| **输出** | 允许访问或拒绝 |
| **验收标准** | - 公开内容所有人可见<br>- 登录内容需登录后可见<br>- VIP 内容需特定角色 |

## 权限设计

### 访问级别定义

| 级别 | 名称 | 说明 |
|------|------|------|
| 0 | 公开 | 所有人可见 (包括访客) |
| 1 | 登录可见 | 已登录用户可见 |
| 2 | VIP/管理员 | 特定角色可见 |

| 角色 | 说明 | 默认访问级别 |
|------|------|--------------|
| guest | 访客 | 0 |
| user | 普通用户 | 1 |
| author | 作者 | 1 |
| admin | 管理员 | 2 |
