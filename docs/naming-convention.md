## 📋 核心命名规则

### 项目名称 vs 技术框架名

**重要:** 必须严格区分项目名称和技术框架名，避免混淆！

#### ❌ 错误用法

```
❌ "OpenClaw 项目" - 这是技术框架名，不是项目名称
❌ "包子铺 Agent" - 这是项目名称，不是技术框架
❌ 混用导致概念混淆
```

#### ✅ 正确用法

```
✅ **项目名称:** 包子铺 (Baozipu / Steamed Bun Shop)
   - 这是一个包子铺业务系统
   - 包括：用户管理、文章管理、订单系统等
   
✅ **技术框架:** OpenClaw
   - 这是 Agent 协作技术框架
   - 包括：多 Agent 通信、任务分发、进度跟踪等
   - 是底层技术支撑平台
```

## ⚠️ 常见错误与纠正

### 错误 1: 项目名称混淆

```markdown
❌ "OpenClaw 需要开发用户认证功能"
   → OpenClaw 是技术框架，不需要业务功能
   
✅ "包子铺项目需要开发用户认证功能"
   → 正确的业务需求表述
```

---

### 错误 2: 技术框架误用

```markdown
❌ "包子铺框架的 Agent 通信协议"
   → 包子铺是业务项目，不是技术框架
   
✅ "OpenClaw 框架的 Agent 通信协议"
   → 正确的技术框架描述
```

---

### 错误 3: 上下文不清

```markdown
❌ "我们项目使用了多 Agent 协作"
   → 哪个项目？不明确
   
✅ "包子铺项目使用 OpenClaw 框架的多 Agent 协作功能"
   → 清晰区分项目和框架
```
