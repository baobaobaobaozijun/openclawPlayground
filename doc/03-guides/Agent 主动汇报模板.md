# Agent 主动汇报模板

**用途:** 标准化 Agent 向 PM 汇报的格式

---

## 📋 汇报类型

### 1. 阻塞上报 🔴

**触发:** 任务阻塞 > 30 分钟

```
【{Agent} - 阻塞 🔴】

任务：{任务 ID/名称}
状态：阻塞
阻塞原因：{详细描述}
已尝试：{已做的尝试}
需要：{PM 需要做什么}
```

**示例:**
```
【酱肉 - 阻塞 🔴】

任务：TASK-017 单元测试框架
状态：阻塞
阻塞原因：Maven 依赖下载失败，错误：Connection timeout (https://repo.maven.apache.org)
已尝试：1. 重试 3 次 2. 切换网络
需要：PM 协调配置 Maven 镜像（阿里云/腾讯云）
```

---

### 2. 任务完成 🟢

**触发:** 任务完成后 5 分钟内

```
【{Agent} - 完成 🟢】

任务：{任务 ID/名称}
状态：完成
交付物：{文件路径/链接}
验证方式：{PM 如何验收}
下一步：{计划做什么}
```

**示例:**
```
【酱肉 - 完成 🟢】

任务：TASK-017 单元测试框架
状态：完成
交付物：code/backend/src/test/ 共 15 个测试文件
验证方式：mvn test 运行，覆盖率 85%
下一步：开始 TASK-018 集成测试
```

---

### 3. 风险预警 🟡

**触发:** 发现可能延期的风险

```
【{Agent} - 风险 🟡】

任务：{任务 ID/名称}
状态：可能延期
风险描述：{具体风险}
影响评估：{可能延期多久}
建议方案：{你的建议}
需要：{PM 决策什么}
```

**示例:**
```
【豆沙 - 风险 🟡】

任务：TASK-008 登录页面开发
状态：可能延期
风险描述：API 文档尚未提供，无法对接登录接口
影响评估：如 16:00 前无文档，将延期至明天
建议方案：先完成 UI 部分，API 对接等文档提供后进行
需要：PM 协调酱肉优先提供 Auth API 文档
```

---

### 4. 进度同步（定时汇报）🟢

**触发:** 每 2 小时一次（10:00 / 12:00 / 14:00 / 16:00 / 18:00）

```
【{Agent} - 进度同步 🟢】

时间：2026-03-23 14:00
当前任务：{任务 ID/名称}
进度：{百分比}%
状态：{🟢正常 | 🟡警告 | 🔴阻塞}
过去 2 小时完成：{具体内容}
接下来 2 小时计划：{具体内容}
阻塞/需要支持：{无/具体描述}
```

**示例:**
```
【酱肉 - 进度同步 🟢】

时间：2026-03-23 14:00
当前任务：TASK-017 单元测试框架
进度：60%
状态：🟢正常
过去 2 小时完成：
  - 完成 UserService 测试（5 个用例）
  - 完成 ArticleService 测试（8 个用例）
接下来 2 小时计划：
  - 完成剩余 Service 层测试
  - 配置覆盖率报告
阻塞/需要支持：无
```

---

### 5. 决策请求 🔴

**触发:** 需要 PM 立即决策

```
【{Agent} - 决策请求 🔴】

背景：{情况描述}
选项 A: {方案 A + 优缺点}
选项 B: {方案 B + 优缺点}
建议：{你的建议}
截止：{最晚决策时间}
```

**示例:**
```
【酸菜 - 决策请求 🔴】

背景：服务器 8080 端口被占用，后端无法启动
选项 A: 停止占用进程（快速，但可能影响其他服务）
选项 B: 修改后端配置使用 8081 端口（安全，但需要改配置）
建议：选项 B，更安全
截止：15:00 前需要决策
```

---

## 📤 发送方式

**使用 `sessions_send` 发送给 PM:**
```
sessions_send(
  agentId="guantang",
  message="{汇报内容}"
)
```

**或在飞书/微信直接发送**
