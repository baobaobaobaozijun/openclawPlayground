## 📋 使用流程

### 场景 1: 收到新需求 → 创建计划

**步骤:**
1. 确定计划信息（编号、名称、优先级、负责人）
2. 运行 `创建计划` 命令
3. 检查生成的文档
4. 开始派发第 1 轮任务

**示例:**
```powershell
npx plan-manager 创建计划 `
  --编号 "006" `
  --名称 "用户中心" `
  --优先级 "P1" `
  --轮次 5 `
  --负责人 "酱肉" `
  --预计耗时 "2h"
```

**参数说明:**
| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--编号` | ✅ | - | 计划编号（3 位数字，如 006） |
| `--名称` | ✅ | - | 计划名称（不含优先级前缀） |
| `--优先级` | ❌ | P1 | P0/P1/P2 |
| `--轮次` | ❌ | 5 | 1-9 |
| `--负责人` | ❌ | 酱肉 | 酱肉/豆沙/酸菜 |
| `--预计耗时` | ❌ | 2h | 如 1h/2h/30m |

**自动完成:**
- ✅ 创建计划文件夹 `plan-{编号}-p{优先级}-{名称}\`
- ✅ 复制 5 个模板文件（00-plan.md 至 04-feishu-logs.md）
- ✅ 填充计划元数据
- ✅ 更新 master-plan-overview.md
- ✅ 创建工作日志
- ✅ Git 提交

---

### 场景 2: 轮次完成 → 更新进度

**步骤:**
1. 等待 Agent 完成回调
2. PM 验证交付物
3. 运行 `更新进度` 命令
4. 派发下一轮任务

**示例:**
```powershell
npx plan-manager 更新进度 `
  --计划编号 "006" `
  --轮次 1 `
  --状态 "completed" `
  --实际耗时 "10m" `
  --备注 "提前完成"
```

**参数说明:**
| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--计划编号` | ✅ | - | 计划编号 |
| `--轮次` | ✅ | - | 轮次号（1-5） |
| `--状态` | ✅ | - | pending/in-progress/completed/timeout/failed |
| `--实际耗时` | ❌ | 自动计算 | 如 10m/15m/1h |
| `--备注` | ❌ | - | 可选备注 |

**自动完成:**
- ✅ 更新 00-plan.md 进度表
- ✅ 更新 01-round-orders.md 添加本轮记录
- ✅ 更新 master-plan-overview.md
- ✅ 创建工作日志
- ✅ Git 提交

---

### 场景 3: 计划完成 → 验收 + 复盘 + 通知

**步骤:**
1. 确认所有轮次完成
2. 运行 `完成计划` 命令
3. 检查复盘报告
4. 发送飞书通知

**示例:**
```powershell
npx plan-manager 完成计划 `
  --计划编号 "006" `
  --状态 "success" `
  --通知方式 "feishu" `
  --通知对象 "ou_2582f0d104a800bec8417c4f6e799c81"
```

**参数说明:**
| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--计划编号` | ✅ | - | 计划编号 |
| `--状态` | ✅ | - | success/partial/failed/timeout |
| `--通知方式` | ❌ | feishu | feishu/none |
| `--通知对象` | ❌ | 从配置读取 | open_id 或 chat_id |

**自动完成:**
- ✅ 更新 00-plan.md 状态为 completed
- ✅ 生成 03-review.md（基于实际数据）
- ✅ 更新 02-verify-list.md 验收结论
- ✅ 生成飞书通知内容
- ✅ 调用 feishu-im-user-message 发送
- ✅ 更新 04-feishu-logs.md
- ✅ 更新 master-plan-overview.md
- ✅ 创建工作日志
- ✅ Git 提交 + 推送

---

### 场景 4: 查看进度 → 列出计划

**步骤:**
1. 运行 `列出计划` 命令
2. 查看整体进度
3. 识别风险计划（进度滞后）

**示例:**
```powershell
# 查看所有计划
npx plan-manager 列出计划

# 只看进行中的计划
npx plan-manager 列出计划 --状态 "in-progress"

# 只看 P0 优先级计划
npx plan-manager 列出计划 --优先级 "P0"
```

**参数说明:**
| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--状态` | ❌ | 全部 | pending/in-progress/completed/failed/timeout |
| `--优先级` | ❌ | 全部 | P0/P1/P2 |

- 完成通知

## ✅ 最佳实践

1. **计划 ID 连续编号** - 从 001 开始递增
2. **优先级前缀** - P0/P1/P2 便于排序
3. **轮次≤5** - 保持计划粒度适中
4. **及时更新** - 每轮完成后立即 `更新进度`
5. **必须复盘** - 每个计划完成后生成 review
6. **命名规范** - 计划文件夹 `plan-{编号}-p{优先级}-{名称}`
7. **文档完整** - 5 个配套文档缺一不可

## 🔧 故障排查

### 问题 1: 创建计划后文件夹为空
**原因:** templates/ 目录不存在  
**解决:** 检查 `skills/plan-manager/templates/` 是否有 5 个模板文件

### 问题 2: 更新进度后 master-plan-overview.md 未更新
**原因:** 文件路径不正确  
**解决:** 确认 `tasks/00-master/master-plan-overview.md` 存在

### 问题 3: 飞书通知发送失败
**原因:** 通知对象不是有效 open_id/chat_id  
**解决:** 检查 recipient 格式（open_id=ou_xxx, chat_id=oc_xxx）

### 问题 4: Git 提交失败
**原因:** 工作区有未提交更改  
**解决:** 先执行 `git add . && git commit -m "xxx"`
